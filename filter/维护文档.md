# 一、 如何“新增”一个主题 (Topic)？
假设运营提出要打击一种新型诈骗：“虚假中奖缴税”。你只需要按顺序做以下 3 步：

第 1 步：在 JSON 里加语料（教大模型认识它）
打开 config_topics_anchors.json，在最下面加一个新 key（比如 "fake_lottery"），写十几句骗子常说的话：

JSON
  "fake_lottery": [
    "恭喜你中了一等奖",
    "您的手机号已被节目组抽中",
    "领奖需要先缴纳个人所得税",
    "这是公证处的领奖保障金",
    "不交税的话奖品将作废"
  ]

# 第 2 步：在 Python 里写打分规则
打开 config_topics.py，找个空白地方，照猫画虎定义这个主题（复制现成的改改名字就行）：

Python
_TOPIC_FAKE_LOTTERY = TopicDefinition(
    topic_id    = "fake_lottery",             # 必须和 json 里的 key 一模一样
    category    = TopicCategory.HIGH_RISK,    # HIGH_RISK 是加分，LOW_VALUE_NOISE 是扣分
    description = "虚假中奖/诱导交税诈骗",
    threshold   = 0.72,                       # 大模型触发阈值，默认写 0.72 即可
    bge_anchors = [],                         # 留空，系统会自动去 json 里抓取
    syntax_rules = [
        # 如果需要加硬正则探针，写在这里；不需要就留空 []
    ],
    scoring_rules = ScoringRules(
        standalone_score = 15,                # 只要大模型命中，直接给 15 分
        standalone_tag   = "has_fake_lottery_intent", # 打上的标签
        matrix_combinations = [
            # 如果你想设置：中奖意图 + 让你转账 = 极高危
            MatrixCombination("has_action_target_triplet", 25, "lottery_transfer_trap"),
        ],
    ),
)
# 第 3 步：注册挂载（激活它）

在 config_topics.py 文件的最底部，找到 TOPIC_REGISTRY 字典，把刚才写的名字加进去：

Python
TOPIC_REGISTRY: dict[str, TopicDefinition] = {
    # ... 其他主题 ...
    "fake_lottery": _TOPIC_FAKE_LOTTERY,  # <--- 加这一行，大功告成！
}
重启服务，新主题就立刻在全链路生效了！

# 二、 如何“暂时屏蔽/下线”一个主题？
这个操作极其简单，千万不要去删除代码，以免以后想恢复时找不回来。

你只需要打开 config_topics.py 的最底部，找到 TOPIC_REGISTRY 字典，在你想屏蔽的主题前面加一个 # 号注释掉即可。

例如，你想暂时屏蔽“毒品黑话”的拦截：

Python
TOPIC_REGISTRY: dict[str, TopicDefinition] = {
    "fraud_jargon":           _TOPIC_FRAUD_JARGON,
    "fraud_object":           _TOPIC_FRAUD_OBJECT,
    "authority_entity":       _TOPIC_AUTHORITY_ENTITY,
    
    # "drug_jargon":            _TOPIC_DRUG_JARGON,   <--- 加个 # 号注释掉
    # "drug_chain":             _TOPIC_DRUG_CHAIN,    <--- 加个 # 号注释掉
    
    "coercive_org_control":   _TOPIC_COERCIVE_ORG_CONTROL,
    # ...
}
原理解析： 因为整个风控系统的雷达、句法提取器和打分器，都是在启动时动态读取 TOPIC_REGISTRY 这个字典的。只要这个字典里没有它，系统就当它完全不存在，既不会去测算，也不会打分，性能还会略微提升。日后想恢复，把 # 删掉重启即可。

# 💡 日常离线维护速查表 (Cheat Sheet)
想改标签名字？ → 改 config_topics.py 里的 standalone_tag 或 bonus_tag。

觉得某个主题太敏感，容易误杀？ → 把它的 standalone_score 调低，或者把 threshold（阈值）从 0.72 调高到 0.75。

发现漏拦了骗子的新话术？ → 直接在 config_topics_anchors.json 对应的主题下，把骗子的新原话复制加进去。

想加个新的脏话让系统一票否决？ → 搜 PROFANITY_REGISTRY，把词加进列表。

想加个绝对底线词（比如恐怖袭击词），一触发直接 100 分阻断？ → 搜 GLOBAL_REDLINE_REGISTRY 加进去。
